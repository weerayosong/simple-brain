# Simple Brain | LLM Wiki (ระบบสมองที่สองด้วย AI)

***"I have Single Brain, i want Second Brain, then i make Simple Brain"***

โปรเจกต์นี้คือ **"ศูนย์บัญชาการความรู้ (Knowledge Control Center)"** ที่ออกแบบมาเพื่อทำงานร่วมกับ AI (Gemini) และระบบ Multi-Agent โดยเฉพาะ จุดประสงค์คือการลดภาระมนุษย์ในการจัดระเบียบข้อมูล และสร้างฐานความรู้ที่ AI สามารถอ่านทำความเข้าใจได้อย่างรวดเร็วและแม่นยำที่สุด

## 🛠️ ความต้องการของระบบ (Prerequisites)
1. **[Obsidian](https://obsidian.md/) (แนะนำอย่างยิ่ง)**: ใช้สำหรับเปิดไฟล์, ดู Graph View (โครงข่ายความรู้), และจัดการ YAML
2. **Git**: ระบบควบคุมเวอร์ชัน สำหรับรันสกิล `/sync`
3. **Gemini IDE / Antigravity**: ตัวรัน AI Agent หลัก

---

## ⚙️ การติดตั้งและเริ่มใช้งาน (Setup & Installation)
เมื่อคุณดึง (Clone) โปรเจกต์นี้มาใช้งานครั้งแรก ให้ทำตามขั้นตอนเหล่านี้เพื่อให้ระบบทำงานได้สมบูรณ์:

1. **ตั้งค่า GitHub Repository ของตัวเอง**:
   - เปิดไฟล์ [`.agents/config.json`](.agents/config.json)
   - เปลี่ยนค่า `github_remote_url` ให้เป็นลิงก์ Repository ของคุณเอง (เช่น `https://github.com/weerayosong/simple-brain.git`)
   - *หมายเหตุ: หากคุณไม่ได้เปลี่ยน ระบบจะดึงข้อมูลนี้ไปตั้งค่า Git Remote เวลารันคำสั่ง `/sync`*
2. **ตั้งเป้าหมายชีวิตปัจจุบันให้ AI รู้**:
   - เปิดไฟล์ [`CONTEXT.md`](CONTEXT.md)
   - อัปเดตเป้าหมายว่าตอนนี้คุณกำลังโฟกัสเรื่องอะไร (เช่น กำลังหางาน, กำลังเรียนภาษา Go) เพื่อให้ AI ตอบคำถามได้ตรงบริบทที่สุด
3. **เปิดโปรเจกต์ด้วย Obsidian**:
   - เลือก Open Folder และชี้มาที่โฟลเดอร์ `simple-brain` นี้

## 🔒 ความเป็นส่วนตัว (Privacy & Syncing)
คุณเลือกใช้งานระบบสมองที่สองนี้ได้ 2 รูปแบบ:
1. **Local Only (เน้นความลับขั้นสุด)**: ข้อมูลถูกเก็บในคอมพิวเตอร์ของคุณเท่านั้น ไม่ต้องตั้งค่า Git ก็ใช้งานฟีเจอร์จัดการความรู้ของ AI ได้ปกติ
2. **Cloud Sync (เน้นแบคอัปกันหาย)**: รันคำสั่ง `/sync` เพื่อให้ AI ช่วยเช็คไฟล์และดัน (Push) ขึ้นไปยัง Private Repository ของคุณบน GitHub

## 📁 โครงสร้างโฟลเดอร์ (Directory Analysis)

| **`/inbox/agent-tasks.md`** | กระดานฝากงานให้ AI เข้ามาทำแบบ Async | อ่านเมื่อถูกเรียกทำงานผ่าน Background หรือ `/schedule` |
| **`/templates`** | เก็บไฟล์ต้นแบบเพื่อให้ AI เขียน Markdown ได้มาตรฐาน | อ่านทุกครั้งก่อนสร้างไฟล์ Wiki, Snippet หรือ Pattern ใหม่ |
| **`/raw`** | กล่องขาเข้าสำหรับข้อมูลดิบ โน้ตมั่วๆ ลิงก์ | อ่าน **เฉพาะตอนสั่ง `Translate Raw`** |
| **`/wiki`** | คลังความรู้หลักที่ AI สกัดและจัดฟอร์แมตแล้ว | **อ่านตลอดเวลา** (แหล่งข้อมูลอ้างอิงหลัก) |
| **`/snippets`** | เก็บโค้ด, Config เด็ดๆ (เช่น Dockerfile) | อ่านเมื่อสั่งให้ **"เขียนโค้ด"** |
| **`/patterns`** | เก็บบันทึกวิธีแก้ปัญหา หรือ Architecture | อ่านเมื่อสั่งให้ **"แก้บั๊ก / วางโครงสร้าง"** |
| **`/archive`** | เก็บไฟล์ดิบที่ประมวลผลเสร็จแล้ว | แทบไม่อ่านเลย (ยกเว้นค้นหาประวัติ) |

*(หมายเหตุ: ไฟล์ตั้งค่าหน้าจอส่วนตัวอย่าง `.obsidian/workspace` จะถูกตั้งค่าให้อยู่ใน `.gitignore` เพื่อไม่ให้ไปรกใน Git)*

## 🚀 ฟีเจอร์ & สกิลหลัก (Skills & Commands)

ระบบนี้ขับเคลื่อนด้วยพลังของ **Gemini Skills** คุณสามารถสั่งงาน AI ด้วยภาษาพูด หรือรันคำสั่งเหล่านี้ได้เลย:

### 1. `Translate Raw` (เปลี่ยนขยะเป็นความรู้)
* **วิธีสั่ง:** *"ช่วยจัดการไฟล์ใน raw ให้หน่อย"* หรือ *"รัน translate"*
* **หน้าที่:** ดึงข้อมูลที่สะเปะสะปะใน `/raw` มาเขียนใหม่ให้เป็นระเบียบลง `/wiki` และย้ายของเดิมไป `/archive`

### 2. `Ingest Codebase` (สกัดความรู้จากโค้ด)
* **วิธีสั่ง:** *"ช่วย Ingest โปรเจกต์ที่ C:/.../my-app ให้หน่อย"*
* **หน้าที่:** สแกนโปรเจกต์โค้ดที่คุณต้องการ และเขียนคู่มือ (Architecture, Components) ลง `/wiki` เพื่อประหยัด Context ให้ AI ตัวอื่น

### 3. `Update Context` (อัปเดตเป้าหมาย)
* **วิธีสั่ง:** *"มา Catch up กันหน่อย"* หรือ *"อัปเดต Context ให้ที"*
* **หน้าที่:** AI จะสัมภาษณ์คุณสั้นๆ 1-2 คำถามว่าช่วงนี้ชีวิตทำอะไรอยู่ จากนั้นจะทำการแก้ไขไฟล์ `CONTEXT.md` ให้อัตโนมัติ

### 4. `Sync Workspace` (ตรวจสอบและแบคอัป)
* **วิธีสั่ง:** `/lint` หรือ `/sync`
* **หน้าที่:** AI จะเช็คความถูกต้องของไฟล์ หากโอเค จะเตรียมตั้งค่า Git อัตโนมัติ (จาก config.json) ร่าง Commit Message และกด Push ขึ้น GitHub เมื่อคุณอนุญาต

### 5. `Weekly Review` (ประเมินรายสัปดาห์)
* **วิธีสั่ง:** *"สรุปงานสัปดาห์นี้ให้หน่อย"* หรือ *"รัน Weekly Review"*
* **หน้าที่:** AI จะไปเช็คประวัติการเพิ่มไฟล์ หรือ Git Commit ย้อนหลัง 7 วัน แล้วสรุปผลงานเทียบกับเป้าหมายใน `CONTEXT.md` พร้อมแนะนำสิ่งทีควรทำสัปดาห์หน้า

### 6. `Ideate / Brainstorm` (ร่างโปรเจกต์จากไอเดียดิบ)
* **วิธีสั่ง:** *"ช่วย Ideate โปรเจกต์ใหม่ให้หน่อย"* หรือ *"ขยายไอเดียแอปใหม่"*
* **หน้าที่:** AI จะสวมบทบาทเป็น Architect และ PM เพื่อแปลงไอเดียสั้นๆ ของคุณให้เป็น Project Plan แบบละเอียด (MVP, Tech Stack, Tasks) และบันทึกลง `/wiki` ให้ทันที พร้อมทำ YAML Routing แบบ `target_agents: [architect, planning]`

### 7. `Refactor Knowledge` (ควบรวมความรู้ที่ซ้ำซ้อน)
* **วิธีสั่ง:** `/refactor` หรือ *"ช่วยรวมไฟล์ snippets ที่ซ้ำกันหน่อย"*
* **หน้าที่:** AI จะสแกนหาไฟล์ใน `/snippets` และ `/patterns` ที่มีเนื้อหาคล้ายกัน นำมายุบรวมเป็น Global Pattern เดียว และอัปเดตลิงก์

### 8. `Generate Index` (สร้างสารบัญอัตโนมัติ)
* **วิธีสั่ง:** *"ช่วยรันสกิล Generate Index ให้หน่อย"*
* **หน้าที่:** AI จะสแกนไฟล์ทั้งหมดในระบบ, ดึง Metadata ออกมา, และสร้างไฟล์ `INDEX.md` (Map of Content) เพื่อจัดหมวดหมู่โปรเจกต์ทั้งหมดให้สวยงามพร้อมลิงก์เชื่อมโยง
