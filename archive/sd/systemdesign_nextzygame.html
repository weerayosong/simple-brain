2. Core Technical Strategies & Standards

2A. Architecture & Code Quality

Clean Architecture

แยก Presentation (Next.js) ออกจาก Application/Domain Logic (NestJS) อย่างเด็ดขาด หน้าบ้านรับหน้าที่แสดงผล การคำนวณและ Business Logic ทั้งหมดจะถูกห่อหุ้มไว้ใน Services ของหลังบ้าน

TypeScript & Minimalist

บังคับใช้ TypeScript Strict Mode ทั้งระบบ และใช้ Native Features (เช่น Fetch API, Tailwind v4 ไร้ Config) เพื่อลด Dependency ขยะและรักษาความคลีนของโปรเจกต์

2B. Performance & Data Strategies

Denormalization & Indexing

จัดเก็บ totalPoints ไว้ในตาราง User โดยตรงเพื่อเลี่ยงคำสั่ง SUM และสร้าง Index ที่ userId เพื่อรองรับการค้นหาประวัติจากข้อมูล 1 ล้านแถวอย่างรวดเร็ว

Offloading to Supabase

ตัดภาระเซิร์ฟเวอร์ด้วยการใช้ฟีเจอร์ Bulk Import CSV ผ่านหน้า Dashboard ของ Supabase โดยตรง เพื่อนำเข้าข้อมูล 1,000,000 แถวอย่างปลอดภัยและรวดเร็ว

High Performance Rendering

ใช้ประโยชน์จาก Vercel Edge Network ควบคู่กับการทำ Responsive Design อย่างประณีต เพื่อรักษาคะแนน PageSpeed (FCP, LCP) ให้อยู่ในเกณฑ์สูงสุด

2C. Reliability & Testing

Global Error Pipeline

จัดการ Error อย่างเป็นระบบ หลังบ้านใช้ Exception Filters (NestJS) คืนค่า HTTP Status ที่ถูกต้อง ส่วนหน้าบ้านดักจับด้วย Try-Catch และแสดงผลผ่าน UI Toast ที่เป็นมิตรกับผู้ใช้

Automated Testing (Jest)

เขียน Unit Test ด้วย Jest ควบคุมคุณภาพของ Business Logic ที่สำคัญ โดยเฉพาะ GameService (การสุ่มคะแนน) และ RewardService (การตรวจสอบ Checkpoint)

Database Constraints

ป้องกันการทุจริตและการรับรางวัลซ้ำซ้อนด้วย @@unique([userId, checkpoint]) ในระดับ Prisma Schema ซึ่งเป็นด่านสุดท้ายที่แข็งแกร่งที่สุดแม้เกิด Race Condition

4. API Specification Contract

Method Endpoint Description & Payload
POST /users/login ตรวจสอบผู้เล่นเก่า/สร้างใหม่
Body: { nickname: string }
POST /game/spin สุ่มคะแนน บันทึกประวัติ และอัปเดตคะแนนรวม
Body: { userId: string }
POST /rewards/claim ตรวจสอบเงื่อนไขและบันทึกการรับรางวัล (Checkpoint)
Body: { userId: string, checkpoint: number }
GET /history/:userId ดึงข้อมูลประวัติผู้เล่นทั่วโลกจากตารางใหญ่ (จำกัด Query ด้วย Pagination)
Query: ?page=1&limit=50
