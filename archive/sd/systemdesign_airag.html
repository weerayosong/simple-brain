2. Core Features & AI Capabilities

Semantic Product Search

ก้าวข้ามการค้นหาแบบ Keyword Match ผู้ใช้สามารถพิมพ์ความต้องการแบบธรรมชาติ (เช่น "หาครีมทาหน้าสำหรับคนเป็นสิวราคาไม่เกิน 20 ดอลลาร์") AI จะวิเคราะห์ความตั้งใจและคัดกรองสินค้าที่ตรงกับบริบทที่สุดมานำเสนอ

RAG-Powered Q&A

ระบบจะตอบคำถามเกี่ยวกับสินค้าโดยอ้างอิงจากข้อมูลโภชนาการ เรตติ้ง หรือคำอธิบายที่ดึงมาจาก DummyJSON เท่านั้น ป้องกันปัญหา AI มั่วข้อมูล (Hallucination) ทำให้แอปมีความน่าเชื่อถือสูง

Structured JSON Rendering

Gemini Flash 3.5 จะถูกบังคับให้ตอบกลับมาเป็นโครงสร้างข้อมูล (Structured Output) เสมอ ทำให้ Frontend สามารถนำข้อมูลไปวาดเป็นการ์ดสินค้า (Product Card) ที่สวยงามได้ทันที ไม่ใช่แค่ข้อความทื่อๆ

Context-Aware Memory

ระบบมีการเก็บ State ประวัติการแชทส่งไปพร้อมกับคำถามใหม่เสมอ ทำให้ผู้ใช้สามารถสั่งงานต่อเนื่องได้ เช่น "เอาตัวเมื่อกี้ แต่ขอราคาถูกกว่านี้" AI จะเข้าใจบริบทต่อเนื่องได้ทันที

4. State Management & Error Pipeline

React State (Frontend)

  • Chat History: ใช้ useState<Message[]> เพื่อเก็บประวัติการสนทนาทั้งหมดใน Session ปัจจุบัน
  • Loading State: แสดง UI ไข่ปลา (Typing Indicator) ระหว่างรอ fetch() โดยผูกกับ isProcessing
  • Auto-Scroll: ใช้ useRef ผูกกับจุดล่างสุดของช่องแชท เพื่อดันหน้าจอลงอัตโนมัติเมื่อมีข้อความใหม่

Error Handling Pipeline

  • AI Rate Limit (429): ดักจับเมื่อ API Gemini เต็มโควต้า และแจ้งเตือนผู้ใช้ให้รอ 1 นาที
  • Fallback Response: หากดึงข้อมูลจาก DummyJSON พลาด ให้ AI ตอบกลับเป็นข้อความขออภัยโดยที่แอปไม่พัง
  • Validation: ป้องกันการส่งข้อความว่างเปล่าจากหน้า UI ก่อนยิง API เสมอ

5. Core TypeScript Interfaces

Interface Structure Purpose
Product { id, title, price, category, thumbnail } โมเดลสินค้า ใช้รับข้อมูลจาก DummyJSON และเรนเดอร์ UI
ChatRequest { message: string, history?: Message[] } ควบคุม Payload ขาเข้าที่ API Endpoint ต้องการ

6. Internal Module Architecture

Frontend Layers (Client)

  • Components UI ล้วนๆ (Dumb Components) เช่น ChatBubble, ProductCard รับ Props มาแสดงผลอย่างเดียว
  • Hooks แยก Logic ออกจาก UI เช่น useChat() จัดการ State ข้อความและการโหลด
  • Services (API Client) ศูนย์รวมการยิง API คุยกับ Backend เช่น chat.service.ts

Backend Layers (Server)

  • Routes กำหนดเส้นทาง API (Endpoints) เช่น POST /api/chat แล้วส่งต่อให้ Controller
  • Controllers รับ Request (ดึง Body/Params) ตรวจสอบความถูกต้อง แล้วเรียกใช้ Services
  • Services (Business Logic) gemini.service.ts จัดการ Prompt AI, dummy.service.ts ดึงข้อมูลสินค้า
