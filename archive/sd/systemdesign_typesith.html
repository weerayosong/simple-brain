2. Core Type Definitions (src/types)

Interface / Type Base / Structure Description
BaseEntity { id, name, era, combatStyle, midichlorian } คุณสมบัติพื้นฐานที่ผู้ใช้พลัง (Force Users) ทุกคนต้องมี
JediEntity extends BaseEntity
side: "Jedi"
{ jediRole, lightsaberColor } 		Interface สำหรับเจได (ใช้ Literal Type กำหนด side เป็น Jedi)
SithEntity extends BaseEntity
side: "Sith"
{ darthTitle, sithRole, corruptionLevel } 		Interface สำหรับซิธ มีฟิลด์เฉพาะเช่น darthTitle และ percent การถูกครอบงำ
ForceEntity JediEntity | SithEntity Discriminated Union: ไทป์หลักที่นำไปใช้เป็น Props ของ Card ควบคุมให้ TS แยกแยะประเภทข้อมูลได้ถูกต้องตามฟิลด์ side

3. Project Structure Breakdown

src/components/

รวบรวม UI ทั้งหมด เน้นเขียนแบบ Dumb Components (รับ Props แสดงผลอย่างเดียว)

  • App.tsx (State Holder)
  • EntityCard.tsx
  • FilterControls.tsx
  • Pagination.tsx

src/utils/

แหล่งรวม Pure Functions ที่แยก Logic การคำนวณออกมาจากไฟล์ UI

  • dataProcessor.ts
    • - filterByCriteria(data, filters)
    • - sortData(data, sortBy)
    • - paginateData(data, page, limit)
